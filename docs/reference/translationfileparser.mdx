title: "translationfileparser"
description: "Strategy-based parser layer for JSON, XLIFF, and PO translation files."
---

`translationfileparser` is a strategy-based parser for local translation files.

## Supported formats

- `.json` with `JSONParser`
- `.xlf` / `.xliff` with `XLIFFParser` (1.2 and 2.x)
- `.po` with `POFileParser`

## Strategy API

- `NewDefaultStrategy()`: pre-registers JSON, XLIFF, and PO parsers.
- `Register(ext, parser)`: add or replace parser implementations by extension.
- `Parse(path, content)`: resolves parser by extension and returns `map[string]string`.

## Parser behavior

### JSON

- Accepts object-shaped JSON.
- Flattens nested objects using dotted keys.
- Rejects non-string leaf values.

### XLIFF

- Key selection order: `id`, then `name`, then `resname`.
- Supports `<trans-unit>` (1.2) and `<unit>` (2.x).
- Uses `<target>` when present, otherwise `<source>`.

### PO

- Maps `msgid` to `msgstr`.
- Supports multiline quoted continuations.
- Uses `msgstr[0]` for plurals.
- Skips header entry (`msgid ""`).
- Ignores comments and `msgctxt` in current behavior.

## Minimal usage

```go
strategy := translationfileparser.NewDefaultStrategy()

values, err := strategy.Parse("lang/fr.xliff", content)
if err != nil {
    return err
}

fmt.Println(values["checkout.submit"])
```
