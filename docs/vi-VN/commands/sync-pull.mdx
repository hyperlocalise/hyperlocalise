<Note>
Tính năng này đang trong giai đoạn thử nghiệm. Các cờ, hình dạng đầu ra và hành vi có thể thay đổi giữa các phiên bản. Sử dụng trong quy trình làm việc được kiểm soát và khóa phiên bản trong CI.

Việc giải quyết xung đột và đồng bộ hóa hợp đồng có thể thay đổi khi các tích hợp TMS phát triển.
</Note>

## Cách sử dụng

```bash
hyperlocalise sync pull [--config <path>] [flags]
```

## Hành vi mặc định an toàn

`sync pull` defaults to dry-run mode. Start by reviewing the report, then rerun with `--dry-run=false`.

## Cờ

- `--config`: config path
- `--locale`: target locale(s), repeatable
- `--dry-run`: chỉ xem (mặc định `true`)
- `--output`: `text` hoặc `json`
- `--fail-on-conflict`: return error when conflicts are detected (default `true`)
- `--apply-curated-over-draft`: allow curated remote values to replace local draft values (default `true`)

## Ví dụ

```bash
hyperlocalise sync pull --locale es-ES --output json
```

Xem [cách giải quyết xung đột](/troubleshooting/conflicts) để biết chiến lược.
