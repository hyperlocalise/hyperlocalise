<Note>
Tính năng này vẫn đang trong giai đoạn thử nghiệm. Các cờ, hình dạng đầu ra và hành vi có thể thay đổi giữa các phiên bản. Sử dụng trong quy trình làm việc được kiểm soát và ghi lại các phiên bản trong CI.

Cấu trúc báo cáo và hành vi chấm điểm có thể thay đổi khi quy trình đánh giá phát triển.
</Note>

Use `eval` commands to benchmark translation quality on a representative eval set and enforce quality thresholds in CI.

## Cách sử dụng

```bash
hyperlocalise eval run --eval-set <path> [flags]
hyperlocalise eval compare --candidate <path> --baseline <path> [flags]
```

## eval chạy

Chạy một hoặc nhiều biến thể thí nghiệm trên bộ dữ liệu đánh giá của bạn.

### Lệnh này làm gì

1. Tải tập dữ liệu eval từ `--eval-set`.
2. Mở rộng các biến thí nghiệm từ các hồ sơ, nhà cung cấp, mô hình và tùy chỉnh prompt đã chọn của bạn.
3. Thực hiện mọi trường hợp đối với mọi biến thể thí nghiệm.
4. Đánh giá mỗi bản dịch bằng cách sử dụng quy tắc heuristic tích hợp.
5. Optionally adds an LLM judge lane when you pass both `--eval-provider` and `--eval-model`.
6. Writes a JSON report when you pass `--output`.
7. In쇄 một bảng tóm tắt ngắn gọn cho mỗi thí nghiệm.

### Cờ

- `--eval-set`: path to eval dataset (`.json`, `.jsonc`, `.csv`) (required)
- `--profile`: profile name override (repeatable)
- `--provider`: ghi đè nhà cung cấp (có thể lặp lại)
- `--model`: ghi đè mô hình (có thể lặp lại)
- `--prompt-file`: ghi đè tệp hướng dẫn
- `--prompt`: inline prompt override (mutually exclusive with `--prompt-file`)
- `--eval-provider`: provider for LLM evaluation
- `--eval-model`: mô hình đánh giá cho LLM
- `--eval-prompt-file`: evaluation prompt file override
- `--eval-prompt`: inline evaluation prompt override (mutually exclusive with `--eval-prompt-file`)
- `--output`: JSON output report path

LLM evaluation mode turns on only when you pass both `--eval-provider` and `--eval-model`.

Your translation flags keep their current meaning. The `--eval-*` flags only control the judge lane.

Trong chế độ đánh giá bằng LLM, các bản dịch tham khảo là tùy chọn. Khi có, người đánh giá sẽ sử dụng chúng làm hướng dẫn về phong cách và giọng văn.

### Các trường trong bảng tóm tắt

- `score`: điểm chất lượng trung bình có trọng số cho thí nghiệm
- `pass_rate`: số lần chạy thành công / tổng số lần chạy cho thí nghiệm
- `placeholder_violations`: count of placeholder integrity hard-fail violations
- `latency_ms`: average latency for the experiment

### Ví dụ

Chạy với các cài đặt mặc định từ cấu hình của bạn và tạo báo cáo:

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.jsonc \
  --output ./artifacts/eval-report.json
```

Thực hiện thử nghiệm với nhiều cấu hình và thay đổi cài đặt cho nhà cung cấp/mô hình:

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.csv \
  --profile default \
  --profile fast \
  --provider openai \
  --provider anthropic \
  --model gpt-4.1-mini \
  --model claude-sonnet-4-5
```

Chạy với tệp cấu hình tùy chỉnh:

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.json \
  --prompt-file ./prompts/translation-eval.txt
```

Chạy với đánh giá bằng LLM bằng lời nhắc giám khảo trực tiếp:

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.json \
  --eval-provider openai \
  --eval-model gpt-4.1-mini \
  --eval-prompt "Score translation quality from 0.0 to 1.0 and explain briefly." \
  --output ./artifacts/eval-report.json
```

Chạy với đánh giá bằng LLM sử dụng tệp hướng dẫn:

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.json \
  --eval-provider anthropic \
  --eval-model claude-sonnet-4-5 \
  --eval-prompt-file ./prompts/eval-judge.txt \
  --output ./artifacts/eval-report.json
```

## eval so sánh

So sánh báo cáo của ứng viên với báo cáo cơ sở.

Sử dụng lệnh này trong CI để ngăn chặn các vấn đề về chất lượng.

The workflow stays the same: run `eval run` first, then run `eval compare`.

### Cờ

- `--candidate`: candidate report JSON path (required)
- `--baseline`: đường dẫn JSON báo cáo cơ sở (bắt buộc)
- `--min-score`: minimum candidate score
- `--max-regression`: maximum allowed score regression from baseline to candidate

### Hành vi của CI

`eval compare` prefers the LLM aggregate score when both reports include a usable LLM judge aggregate. Otherwise, it falls back to the heuristic weighted score.

Lệnh này sẽ kết thúc với lỗi khi:

- candidate score is below `--min-score`, or
- score regression exceeds `--max-regression`.

### Ví dụ

So sánh các báo cáo và in chỉ các giá trị tóm tắt:

```bash
hyperlocalise eval compare \
  --candidate ./artifacts/eval-candidate.json \
  --baseline ./artifacts/eval-baseline.json
```

Fail CI if the candidate score drops below `0.82` or regresses by more than `0.02`:

```bash
hyperlocalise eval compare \
  --candidate ./artifacts/eval-candidate.json \
  --baseline ./artifacts/eval-baseline.json \
  --min-score 0.82 \
  --max-regression 0.02
```
