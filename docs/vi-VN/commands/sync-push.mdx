<Note>
Tính năng này đang trong giai đoạn thử nghiệm. Các cờ, hình dạng đầu ra và hành vi có thể thay đổi giữa các phiên bản. Sử dụng trong quy trình làm việc được kiểm soát và khóa phiên bản trong CI.

Việc giải quyết xung đột và các hợp đồng đồng bộ có thể thay đổi khi tích hợp TMS phát triển.
</Note>

## Sử dụng

```bash
hyperlocalise sync push [--config <path>] [flags]
```

## Hành vi mặc định an toàn

`sync push` defaults to dry-run mode and conflict failure.

## Cờ

- `--config`: config path
- `--locale`: vùng/khu vực đích (có thể lặp lại)
- `--dry-run`: xem trước (mặc định `true`)
- `--output`: `text` hoặc `json`
- `--fail-on-conflict`: return error when conflicts are detected (default `true`)
- `--force-conflicts`: allow overwrite for mismatch cases where policy allows (default `false`)

## Ví dụ

```bash
hyperlocalise sync push --locale es-ES --dry-run=false
```

See [conflict handling](/troubleshooting/conflicts) before using `--force-conflicts`.
