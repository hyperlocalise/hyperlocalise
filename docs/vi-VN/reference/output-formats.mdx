title: "Định dạng đầu ra"
description: "Tài liệu tham khảo về định dạng đầu ra của các lệnh run, status và sync."
---

## `run` output (`text`)

Các trường tóm tắt chính:

- `planned_total`
- `skipped_by_lock`
- `executable_total`
- `succeeded`
- `failed`
- `persisted_to_lock`

Định dạng dòng lỗi:

```text
failure target=<path> key=<key> reason=<reason>
```

## `status` output (`csv`)

Cột tiêu đề:

```csv
key,namespace,locale,status,origin,state
```

Chế độ tương tác:

- `hyperlocalise status --output csv -i` hiển thị bảng điều khiển TTY để phân loại thủ công.
- CSV vẫn là định dạng chuẩn chính cho máy tính-định dạng dễ đọc cho CI và tự động hóa.

## `sync` đầu ra (`text` hoặc `json`)

Các trường tóm tắt văn bản:

- `action`
- `creates`
- `updates`
- `unchanged`
- `conflicts`
- `applied`
- `skipped`
- `warnings`

Sử dụng đầu ra JSON khi tích hợp với các bước kiểm tra CI.


## Tệp dịch CSV

Khi `run` viết `.csv` mục tiêu bản dịch, thứ tự hàng đầu ra được xác định (deterministic), và các hàng hiện có giữ nguyên thứ tự ban đầu.
