## `run` output (`text`)

Các trường tóm tắt chính:

- `planned_total`
- `skipped_by_lock`
- `executable_total`
- `succeeded`
- `failed`
- `persisted_to_lock`

Định dạng dòng lỗi:

```text
failure target=<path> key=<key> reason=<reason>
```

## `status` đầu ra (`csv`)

Tiêu đề cột:

```csv
key,namespace,locale,status,origin,state
```

Chế độ tương tác:

- `hyperlocalise status --output csv -i` renders a TTY dashboard for manual triage.
- CSV vẫn là định dạng có thể đọc được bằng máy phổ biến nhất cho CI và tự động hóa.

## `sync` output (`text` or `json`)

Các trường tóm tắt văn bản:

- `action`
- `creates`
- `updates`
- `unchanged`
- `conflicts`
- `applied`
- `skipped`
- `warnings`

Sử dụng đầu ra JSON khi tích hợp với kiểm tra CI.


## Tệp dịch thuật CSV

When `run` writes `.csv` translation targets, output row order is deterministic for newly appended keys (sorted by key), and existing rows keep their original order.
