## Vòng lặp được khuyến nghị

1. `hyperlocalise run --dry-run`
2. kiểm tra danh sách và số lượng công việc
3. `hyperlocalise run`
4. kiểm tra lỗi và chạy lại nếu cần

## Hành vi của file khóa

`.hyperlocalise.lock.json` tracks completed tasks to avoid repeated work.

Khi bạn thay đổi nội dung nguồn hoặc cấu hình ánh xạ, hãy chạy lại và xác minh xem có nhiệm vụ mới nào được lên lịch hay không.

Sử dụng [Hợp đồng file khóa](/reference/lockfile-contract) để biết chi tiết về sơ đồ và hướng dẫn đặt lại.

## Thực hành tốt

- giữ nguyên chế độ chạy thử trong CI để có khả năng quan sát
- sử dụng ánh xạ tệp có tính quyết định
- tránh chỉnh sửa thủ công trạng thái khóa

## Markdown trong quy trình làm việc cục bộ

If your bucket mappings point to `.md` or `.mdx` files, `hyperlocalise run --dry-run` still plans tasks from Markdown sources.

When you run without `--dry-run`, output keeps Markdown and MDX layout stable while translating extracted prose nodes.
