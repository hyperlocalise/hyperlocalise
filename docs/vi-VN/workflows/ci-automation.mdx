## Tiêu chuẩn tiêm bí mật CI

For remote adapters, keep secrets in your CI secret manager and inject them only in jobs that call `hyperlocalise sync`.

- Commit only `*Env` variable names in `i18n.jsonc`.
- Tiêm giá trị token vào thời chạy thông qua biến môi trường cấp công việc.
- Sử dụng thông tin đăng nhập riêng biệt cho việc kiểm tra chỉ đọc so với các công việc đồng bộ cho phép ghi.
- Tránh in các biến môi trường trong các bước của pipeline.

## Các kiểm tra được đề xuất

### 1. Cấu hình + lập kế hoạch kiểm thử khói

```bash
hyperlocalise run --dry-run
```

### 2. Tóm tắt trạng thái

```bash
hyperlocalise status --output csv
```

Lưu trữ tệp CSV làm một phần của dự án để theo dõi xu hướng.

### 3. Cổng xung đột đồng bộ

`sync pull` and `sync push` default to `--dry-run=true`, which is a safe CI baseline.

```bash
hyperlocalise sync pull --output json --fail-on-conflict --dry-run=true
hyperlocalise sync push --output json --fail-on-conflict --dry-run=true
```

Use `--dry-run=false` only in jobs that are explicitly authorized to write local files or remote systems.

## Ví dụ về công việc đồng bộ theo lịch trình

Sử dụng các ví dụ GitHub Actions này làm cơ sở. Điều chỉnh tên và quy tắc phê duyệt để phù hợp với nền tảng CI của bạn.

### Đồng bộ theo lịch (chỉ đọc, cài đặt mặc định an toàn)

```yaml
name: tms-pull-sync

on:
  schedule:
    - cron: "0 */6 * * *"
  workflow_dispatch:

jobs:
  pull-sync:
    runs-on: ubuntu-latest
    permissions:
      contents: read
    env:
      SMARTLING_USER_SECRET: ${{ secrets.SMARTLING_USER_SECRET }}
      LOKALISE_API_TOKEN: ${{ secrets.LOKALISE_API_TOKEN }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
      - run: go run ./apps/cli sync pull --output json --fail-on-conflict --dry-run=true
```

Để áp dụng các thay đổi từ xa đối với các tệp cục bộ trong CI, hãy chuyển sang:

```bash
go run ./apps/cli sync pull --output json --fail-on-conflict --dry-run=false
```

### Đẩy đồng bộ theo lịch (bật, đang hoạt động)

```yaml
name: tms-push-sync

on:
  schedule:
    - cron: "30 2 * * *"
  workflow_dispatch:

jobs:
  push-sync:
    runs-on: ubuntu-latest
    environment: production
    permissions:
      contents: read
    env:
      SMARTLING_USER_SECRET: ${{ secrets.SMARTLING_USER_SECRET }}
      LOKALISE_API_TOKEN: ${{ secrets.LOKALISE_API_TOKEN }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
      - run: go run ./apps/cli sync push --output json --fail-on-conflict --dry-run=false
```

Để thực hiện kiểm tra không làm thay đổi dữ liệu, hãy sử dụng:

```bash
go run ./apps/cli sync push --output json --fail-on-conflict --dry-run=true
```

Đối với các công việc cho phép ghi, hãy ưu tiên sử dụng môi trường được bảo vệ, các cơ chế kiểm soát và thông tin xác thực dịch vụ có quyền hạn tối thiểu.
