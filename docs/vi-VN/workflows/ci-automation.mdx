## Tiêu chuẩn tiêm bí mật CI

Đối với các adapter từ xa, hãy lưu các bí mật trong trình quản lý bí mật CI của bạn và chỉ đưa chúng vào các job có gọi `hyperlocalise sync`.

- Chỉ commit `*Env` tên biến trong `i18n.jsonc`.
- Chèn giá trị token trong thời gian chạy thông qua job-biến môi trường cấp độ.
- Sử dụng thông tin đăng nhập riêng cho quyền đọc-chỉ kiểm tra so với ghi-đã bật các tác vụ đồng bộ hóa.
- Tránh in các biến môi trường trong các bước pipeline.

## Các kiểm tra được đề xuất

### 1. Cấu hình + lập kế hoạch kiểm thử khói

```bash
hyperlocalise run --config i18n.jsonc --dry-run
```

### 2. Ảnh chụp nhanh trạng thái

```bash
hyperlocalise status --config i18n.jsonc --output csv
```

Lưu tệp CSV dưới dạng artifact của bản dựng để theo dõi xu hướng.

### 3. Cổng xung đột đồng bộ hóa

`sync pull` và `sync push` mặc định là `--dry-run=true`, đây là đường cơ sở CI an toàn.

```bash
hyperlocalise sync pull --config i18n.jsonc --output json --fail-on-conflict --dry-run=true
hyperlocalise sync push --config i18n.jsonc --output json --fail-on-conflict --dry-run=true
```

Sử dụng `--dry-run=false` chỉ trong các công việc được ủy quyền rõ ràng để ghi vào tệp cục bộ hoặc hệ thống từ xa.

## Ví dụ về tác vụ đồng bộ theo lịch

Sử dụng các ví dụ GitHub Actions này làm cơ sở. Điều chỉnh tên bí mật và quy tắc phê duyệt cho phù hợp với nền tảng CI của bạn.

### Đồng bộ kéo theo lịch trình (đọc-chỉ, mặc định an toàn)

```yaml
name: tms-pull-sync

on:
  schedule:
    - cron: "0 */6 * * *"
  workflow_dispatch:

jobs:
  pull-sync:
    runs-on: ubuntu-latest
    permissions:
      contents: read
    env:
      SMARTLING_USER_SECRET: ${{ secrets.SMARTLING_USER_SECRET }}
      LOKALISE_API_TOKEN: ${{ secrets.LOKALISE_API_TOKEN }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
      - run: go run . sync pull --config i18n.jsonc --output json --fail-on-conflict --dry-run=true
```

Để áp dụng các thay đổi từ xa cho các tệp cục bộ trong CI, hãy chuyển sang:

```bash
go run . sync pull --config i18n.jsonc --output json --fail-on-conflict --dry-run=false
```

### Đồng bộ đẩy theo lịch trình (viết-đã bật, đang hoạt động)

```yaml
name: tms-push-sync

on:
  schedule:
    - cron: "30 2 * * *"
  workflow_dispatch:

jobs:
  push-sync:
    runs-on: ubuntu-latest
    environment: production
    permissions:
      contents: read
    env:
      SMARTLING_USER_SECRET: ${{ secrets.SMARTLING_USER_SECRET }}
      LOKALISE_API_TOKEN: ${{ secrets.LOKALISE_API_TOKEN }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
      - run: go run . sync push --config i18n.jsonc --output json --fail-on-conflict --dry-run=false
```

Đối với một không-bước xác thực có thay đổi, sử dụng:

```bash
go run . sync push --config i18n.jsonc --output json --fail-on-conflict --dry-run=true
```

Để ghi-bật các công việc, ưu tiên môi trường được bảo vệ, cổng phê duyệt và tối thiểu-thông tin xác thực của dịch vụ đặc quyền.
