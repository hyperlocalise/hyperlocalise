---

## Tổng quan

Sử dụng action ở thư mục gốc của kho lưu trữ khi bạn muốn một kiểm tra pull request thất bại nếu có sai lệch bản địa hóa.

Hành động chạy `hyperlocalise run --dry-run`, truyền đầu ra CLI tới nhật ký quy trình làm việc và tải lên hai artifact khi được bật:

- `drift-report.json`
- `drift-summary.txt`

## Quy trình ví dụ

```yaml
name: localization-drift

on:
  pull_request:

jobs:
  drift:
    runs-on: ubuntu-latest
    permissions:
      contents: read
    steps:
      - uses: actions/checkout@v4

      - uses: ./
        with:
          check: drift
          config-path: i18n.yml
          hyperlocalise-version: latest
          fail-on-drift: true
          upload-artifact: true
```

Đối với các kiểm tra pull request, chế độ `check` có thể lấy diff của pull request trên GitHub và chuyển nó cho `hyperlocalise check --diff-stdin`. Điều này giới hạn các phát hiện và chú thích nội dòng vào các khóa dịch đã thay đổi cho các tệp `.json`, `.jsonc` và `.arb` được hỗ trợ.

```yaml
name: localization-check

on:
  pull_request:

jobs:
  check:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: read
    steps:
      - uses: actions/checkout@v4

      - uses: hyperlocalise/hyperlocalise@v1
        with:
          check: check
          config-path: i18n.yml
          github-diff: true
          fail-on-findings: true
```

Các kho lưu trữ bên ngoài có thể sử dụng tham chiếu di động `v1` cho bản phát hành v1 ổn định mới nhất. Ghim một thẻ chính xác nếu bạn muốn kiểm soát thay đổi chặt chẽ hơn.

```yaml
- uses: hyperlocalise/hyperlocalise@v1
  with:
    check: drift
```

## Đầu vào

- `check`: kiểm tra tên. Các giá trị được hỗ trợ là `drift` và `check`.
- `config-path`: đường dẫn đến tệp cấu hình. Mặc định là `i18n.yml`.
- `working-directory`: thư mục làm việc cho lệnh. Mặc định là `.`.
- `hyperlocalise-version`: phiên bản phát hành để cài đặt. Mặc định là `latest`.
- `fail-on-drift`: làm cho quy trình làm việc thất bại khi phát hiện độ lệch. Mặc định là `true`.
- `fail-on-findings`: làm cho quy trình thất bại khi phát hiện các kết quả kiểm tra. Mặc định là `true`.
- `github-diff`: trong chế độ `check` trên pull request, lấy diff của GitHub PR và chuyển nó tới `hyperlocalise check --diff-stdin`. Mặc định là `false`.
- `upload-artifact`: tải lên các tệp JSON và văn bản. Mặc định là `true`.

## Phát hiện trôi dạt

Hành động coi một trong hai điều kiện này là lệch chuẩn:

- một hoặc nhiều tác vụ dịch có thể thực thi trong báo cáo chạy thử
- một hoặc nhiều ứng viên loại bỏ trong báo cáo chạy thử

Lỗi lệnh vẫn làm cho hành động thất bại ngay cả khi cưỡng chế drift bị tắt.

## Hiện vật

`drift-summary.txt` là tài liệu phân loại nhanh. Nó ghi lại:

- đường dẫn cấu hình được sử dụng
- mã thoát của CLI
- liệu có phát hiện sự trôi dạt hay không
- các đường dẫn mục tiêu bị ảnh hưởng
- các ngôn ngữ/miền mục tiêu bị ảnh hưởng
- các khóa mục bị ảnh hưởng khi có sẵn

Nếu CLI thất bại trước khi ghi JSON, hành động vẫn ghi một hiện vật văn bản tối thiểu khi có thể.
