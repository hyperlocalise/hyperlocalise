## Tổng quan tệp

`i18n.yml` điều khiển locale, ánh xạ tệp, hồ sơ mô hình, và các cấu hình nâng cao tùy chọn. `i18n.jsonc` vẫn được hỗ trợ để tương thích ngược.

## Cấu hình khởi đầu

```yaml
locales:
  source: en-US
  targets:
    - es-ES

buckets:
  ui:
    files:
      - from: packages/ui/lang/{{source}}.json
        to: apps/web/lang/{{target}}.json

llm:
  profiles:
    default:
      provider: openai
      model: gpt-5.2
```

Đây là cấu trúc được tạo bởi `hyperlocalise init`.

Khi bỏ qua `groups`, Hyperlocalise sẽ tự động dùng một nhóm `default` ngầm với:

- tất cả locale trong `locales.targets`
- tất cả bucket trong `buckets`

## Locales

- `locales.source`: locale nguồn.
- `locales.targets`: danh sách locale đích.
- `locales.fallbacks` (tùy chọn): thứ tự fallback cho từng locale.

## Buckets

`buckets` ánh xạ tệp nguồn sang mẫu đường dẫn đầu ra.

Mỗi ánh xạ tệp dùng:

- `from`: mẫu đường dẫn nguồn
- `to`: mẫu đường dẫn đích

Dùng `{{source}}`, `{{target}}`, và `{{localeDir}}` trong template. `{{localeDir}}` sẽ là chuỗi rỗng khi target trùng source, và là target locale khi khác nhau.

### Mẫu ánh xạ CSV

Dùng bucket để mô hình hóa luồng CSV.

Tệp theo từng locale:

```json
{
  "from": "translations/en.csv",
  "to": "translations/[locale].csv"
}
```

Tệp đa locale dùng chung:

```json
{
  "from": "translations/catalog.csv",
  "to": "translations/catalog.csv"
}
```

Với CSV đa locale dùng chung, hãy giữ một cột khóa và một cột target locale để `run` có thể cập nhật ổn định.

## Groups

`groups.<name>` định nghĩa phần nào sẽ được chạy cùng nhau.

- `targets`: tập con của `locales.targets`
- `buckets`: danh sách tên bucket

`groups` là tùy chọn cho cấu hình cơ bản. Chỉ thêm khi bạn muốn tách execution theo locale hoặc theo tập tệp.

## Hồ sơ LLM

Các trường trong `llm.profiles.<name>`:

- `provider`: `openai`, `azure_openai`, `anthropic`, `gemini`, `bedrock`, `lmstudio`, `groq`, `mistral`, hoặc `ollama`
- `model`: model id của nhà cung cấp
- `system_prompt` (tùy chọn): template system message
- `user_prompt` (tùy chọn): template user message
- `prompt` (đã deprecated, tùy chọn): template cũ dùng làm system fallback

Biến có sẵn trong prompt:

- `{{source}}`
- `{{target}}`
- `{{input}}`

## Rules

`llm.rules` chọn profile theo từng group.

- `priority`: số lớn hơn sẽ thắng
- `group`: tên group
- `profile`: tên profile

`llm.rules` là tùy chọn. Nếu không có rule khớp, Hyperlocalise sẽ dùng `llm.profiles.default`.

## Cache

`cache` cấu hình hành vi L1/L2 cục bộ.

- `enabled` (tùy chọn): bật dịch vụ cache
- `db_path` (tùy chọn): đường dẫn sqlite cho cache
- `sqlite` (tùy chọn): cấu hình kết nối
- `l1.max_items` (tùy chọn): giới hạn eviction cho L1
- `l2.enabled` (tùy chọn): bật tra cứu/upsert translation memory
- `l2.auto_accept_threshold` (tùy chọn): ngưỡng fuzzy auto-accept trong `[0,1]`, mặc định `0.90`

TM metadata enums:

- provenance: `curated` | `draft` | `llm` | `tms` | `unknown`
- source: `run` | `sync_pull` | `sync_push` | `manual` | `import` | `legacy` | `unknown`

## Ví dụ nâng cao

```yaml
locales:
  source: en-US
  targets:
    - fr-FR
    - es-ES
  fallbacks:
    fr-FR:
      - es-ES
      - en-US

buckets:
  ui:
    files:
      - from: packages/ui/lang/{{source}}.json
        to: apps/web/lang/{{target}}.json
  docs:
    files:
      - from: docs/{{source}}/**/*.mdx
        to: docs/{{target}}/**/*.mdx

groups:
  app:
    targets:
      - fr-FR
      - es-ES
    buckets:
      - ui
  docs:
    targets:
      - fr-FR
    buckets:
      - docs

llm:
  profiles:
    default:
      provider: openai
      model: gpt-5.2
    docs_review:
      provider: openai
      model: gpt-5.2
      system_prompt: You are a senior technical translator.
      user_prompt: |-
        Translate from {{source}} to {{target}}.

        Keep Markdown and MDX syntax intact.

        {{input}}
  context_memory:
    provider: openai
    model: gpt-5.2
  rules:
    - priority: 100
      group: docs
      profile: docs_review

cache:
  enabled: true
  db_path: .hyperlocalise/cache/cache.sqlite
  sqlite:
    max_open_conns: 1
    max_idle_conns: 1
    conn_max_lifetime_seconds: 300
  l1:
    enabled: true
    max_items: 50000
  l2:
    enabled: true
    auto_accept_threshold: 0.9
  rag:
    enabled: false
    top_k: 5
```

## Tạo starter

```bash
hyperlocalise init
```

Sau đó chỉnh sửa `i18n.yml` đã tạo cho dự án của bạn.

## Mẹo kiểm tra

- Nếu bỏ qua `groups`, mọi bucket sẽ chạy cho mọi target locale.
- Giữ group targets nằm trong `locales.targets`.
- Giữ group buckets khớp với các khóa trong `buckets`.
- Giữ tên profile nhất quán giữa `profiles` và `rules`.
