---
title: "Smartling adapter"
description: "Smartling adapter configuration and sync behavior."
---

The Smartling adapter implements `sync pull` and `sync push` through Smartling Auth and Strings APIs.

## Sync behavior

- `Pull`: list translations for selected locales and map them to normalized entries.
- `Push`: upsert non-empty local entries keyed by `key + context + locale`.

## Secret resolution

Smartling authentication secret lookup uses this order:

1. Environment variable named by `storage.config.userSecretEnv`.
2. `SMARTLING_USER_SECRET`.

The adapter rejects inline `userSecret` in config. Use environment variables only.

## Config

```jsonc
{
  "storage": {
    "adapter": "smartling",
    "config": {
      "projectID": "your-project-id",
      "userIdentifier": "your-user-identifier",
      "userSecretEnv": "SMARTLING_USER_SECRET",
      "targetLanguages": ["fr", "de"],
      "timeoutSeconds": 30
    }
  }
}
```

Supported config fields:

- `projectID` (required)
- `userIdentifier` (required)
- `userSecretEnv` (optional, default `SMARTLING_USER_SECRET`)
- `targetLanguages` (optional)
- `timeoutSeconds` (optional, default `30`)

## Behavior notes

- Empty locale or value entries are skipped on pull and push.
- Context is supported and mapped to Smartling instruction text.
- Namespace mapping is local-only.
- Conflict resolution is handled by sync service logic.
