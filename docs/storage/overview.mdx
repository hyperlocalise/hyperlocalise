title: "Storage overview"
description: "Normalized translation entry model, adapter contract, and sync conflict behavior."

This storage model is the contract between local translation files, remote adapters, and sync orchestration.

## Core model

A translation entry uses this identity:

- `key`
- `context`
- `locale`

Relevant fields include:

- `value`
- `provenance` (`origin`, `state`, `updated_at`, `updated_by`)
- `remote` metadata (`adapter`, `revision`, `updated_at`)

## Why sidecar metadata exists

Local values remain in normal translation files (for example `lang/fr.json`).
Operational metadata is stored separately (for example `lang/fr.meta.json`) to avoid mixing translation content with provenance state.

## Adapter interface

Remote providers map API payloads to normalized entries through an adapter contract:

- `Pull(...)`
- `Push(...)`
- `Name()`
- `Capabilities()`

Conflict decisions are centralized in sync service logic, not adapter code.

## LLM to curation flow

1. Generate local drafts and mark metadata as `origin=llm`, `state=draft`.
2. Optionally push drafts to remote storage.
3. Human translators curate in remote TMS.
4. Pull curated values back into local files and metadata.

## Pull conflict behavior

- Missing local key with remote value: create locally.
- Equal local and remote values: unchanged.
- Local `llm/draft` differs from remote curated value: update local.
- Local `curated` differs from remote value: conflict.
- Local provenance missing and value differs: conflict.

## Push conflict behavior

- Missing remote key: create candidate.
- Equal local and remote values: unchanged.
- Local draft differs from remote curated value: conflict by default.
- Unknown local provenance with mismatch: conflict by default.
- `--force-conflicts` can allow selected overwrite candidates.

## Operational flags

- `--dry-run`: compute and report only.
- `--fail-on-conflict`: return error when conflicts are present.
- `--apply-curated-over-draft`: allow curated pull-overwrites for local drafts.
- `--force-conflicts`: allow push overwrite candidates where policy permits.
