## CI 密钥注入标准

For remote adapters, keep secrets in your CI secret manager and inject them only in jobs that call `hyperlocalise sync`.

- Commit only `*Env` variable names in `i18n.jsonc`.
- 通过作业级别的环境变量在运行时注入令牌值。
- 使用不同的凭据，用于只读检查和具有写入功能的同步任务。
- 避免在管道步骤中打印环境变量。

## 建议检查

### 1. 配置 + 规划冒烟测试

```bash
hyperlocalise run --dry-run
```

### 2. 状态快照

```bash
hyperlocalise status --output csv
```

将 CSV 数据存储为构建工件，用于趋势跟踪。

### 3. 同步冲突栅

`sync pull` 和 `sync push` 默认为 `--dry-run=true`，后者是一个安全的 CI 基线。

```bash
hyperlocalise sync pull --output json --fail-on-conflict --dry-run=true
hyperlocalise sync push --output json --fail-on-conflict --dry-run=true
```

Use `--dry-run=false` only in jobs that are explicitly authorized to write local files or remote systems.

## 计划同步任务示例

请将这些 GitHub Actions 示例作为基准。 根据您的 CI 平台，调整密钥名称和审批规则。

### 计划同步（只读，安全默认）

```yaml
name: tms-pull-sync

on:
  schedule:
    - cron: "0 */6 * * *"
  workflow_dispatch:

jobs:
  pull-sync:
    runs-on: ubuntu-latest
    permissions:
      contents: read
    env:
      SMARTLING_USER_SECRET: ${{ secrets.SMARTLING_USER_SECRET }}
      LOKALISE_API_TOKEN: ${{ secrets.LOKALISE_API_TOKEN }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
      - run: go run . sync pull --output json --fail-on-conflict --dry-run=true
```

为了在 CI 中将远程更改应用到本地文件，请切换到：

```bash
go run . sync pull --output json --fail-on-conflict --dry-run=false
```

### 计划性推送同步（已启用，处于活动状态）

```yaml
name: tms-push-sync

on:
  schedule:
    - cron: "30 2 * * *"
  workflow_dispatch:

jobs:
  push-sync:
    runs-on: ubuntu-latest
    environment: production
    permissions:
      contents: read
    env:
      SMARTLING_USER_SECRET: ${{ secrets.SMARTLING_USER_SECRET }}
      LOKALISE_API_TOKEN: ${{ secrets.LOKALISE_API_TOKEN }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
      - run: go run . sync push --output json --fail-on-conflict --dry-run=false
```

对于不修改数据的验证，请使用：

```bash
go run . sync push --output json --fail-on-conflict --dry-run=true
```

对于允许写入的作业，建议使用受保护的环境、审批流程和最小权限的服务凭据。
