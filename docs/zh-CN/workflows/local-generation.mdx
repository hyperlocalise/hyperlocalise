## 推荐循环

1. `hyperlocalise run --dry-run`
2. 检查任务列表和计数
3. `hyperlocalise run`
4. 检查失败并在需要时重新运行

## 锁文件行为

`.hyperlocalise.lock.json` 跟踪已完成的任务，以避免重复工作。

当你更改源内容或配置映射时，请重新运行并确认是否计划了新的任务。

使用 [Lockfile 合同](/reference/lockfile-contract) 获取模式详细信息和重置指南。

## 良好实践

- 在 CI 中保留 dry-run 以便可见性
- 使用确定性的文件映射
- 避免对锁定状态进行手动编辑

## 本地工作流中的 Markdown

如果您的桶映射指向`.md`或`.mdx`文件，`hyperlocalise run --dry-run`仍会根据 Markdown 源规划任务。

当您在不使用`--dry-run`的情况下运行时，输出会在翻译提取的散文节点的同时，保持 Markdown 和 MDX 布局稳定。
