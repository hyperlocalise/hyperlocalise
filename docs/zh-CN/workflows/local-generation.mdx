## 推荐循环

1. `hyperlocalise run --dry-run`
2. 检查任务列表及数量统计
3. `hyperlocalise run`
4. 查看失败项，并在需要时重新运行

## 锁定-文件行为

`.hyperlocalise.lock.json` 跟踪已完成的任务，以避免重复工作。

当你更改源内容或配置映射时，重新-运行并验证是否已计划新任务。

使用 [锁文件契约](/reference/lockfile-contract) 有关架构详细信息和重置指南。

## 良好实践

- 保持干燥-在 CI 中运行以提高可见性
- 使用确定性的文件映射
- 避免手动编辑锁定状态

## 本地工作流程中的 Markdown

如果您的存储桶映射指向 `.md` 或 `.mdx` 文件， `hyperlocalise run --dry-run` 仍然从 Markdown 源规划任务。

当你在没有的情况下运行时 `--dry-run`，同时在翻译提取的正文节点时，输出会保持 Markdown 和 MDX 布局稳定。
