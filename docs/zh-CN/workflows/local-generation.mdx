## 推荐循环

1. `hyperlocalise run --dry-run`
2. 检查任务列表和计数
3. `hyperlocalise run`
4. 审查失败并根据需要重新运行

## 锁文件行为

`.hyperlocalise.lock.json` tracks completed tasks to avoid repeated work.

当您更改源内容或配置映射时，请重新运行并验证是否已计划新的任务。

使用 [锁文件合约](/reference/lockfile-contract)] 以获取模式详细信息和重置指南。

## 良好实践

- 在 CI 中保持 dry-run 状态以获得可见性
- 使用确定性文件映射
- 避免手动修改锁状态

## 本地工作流程中的 Markdown

如果您的桶映射指向`.md`或`.mdx`文件，则`hyperlocalise run --dry-run`仍然会从 Markdown 来源计划任务。

When you run without `--dry-run`, output keeps Markdown and MDX layout stable while translating extracted prose nodes.
