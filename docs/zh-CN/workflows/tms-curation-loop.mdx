随着同步语义和冲突策略不断成熟，工作流程指导可能会发生变化。
</Note>

## 端到端序列

1. 在本地生成草稿，使用 `run`
2. 推送草稿并带有`sync push --dry-run=false`
3. 翻译人员在 POEditor/Lokalise/Crowdin 中整理
4. 拉取精选更新，使用`sync pull --dry-run=false`
5. 验证 `status`

## 决策点

- 为了安全起见，请保留 `--fail-on-conflict=true`。
- 在下拉刷新时使用 `--apply-curated-over-draft`，当经过整理的数据应当优先时。
- 仅在查看冲突后，才在推送时使用 `--force-conflicts`。

## 可审计性

使用 JSON 输出以便机器可读的审查：

```bash
hyperlocalise sync pull --output json
hyperlocalise sync push --output json
```
