<Note>
此功能为实验性功能。标志、输出形状和行为可能会在不同版本之间发生变化。请在受控的工作流程中使用，并在 CI 中固定版本。

随着同步语义和冲突策略不断成熟，工作流指南可能会发生变化。
</Note>

## 结束-到-结束序列

1. 使用在本地生成草稿 `run`
2. 推送草稿，使用 `sync push --dry-run=false`
3. 译员在 POEditor/Lokalise/Crowdin 中整理
4. 拉取精选更新，使用 `sync pull --dry-run=false`
5. 使用进行验证 `status`

## 决策点

- 保留 `--fail-on-conflict=true` 为安全起见。
- 使用 `--apply-curated-over-draft` 在拉取时，当应以精选数据为准。
- 使用 `--force-conflicts` 仅在审查冲突后才推送。

## 可审计性

使用 JSON 输出供机器使用-可读性审查：

```bash
hyperlocalise sync pull --output json
hyperlocalise sync push --output json
```
