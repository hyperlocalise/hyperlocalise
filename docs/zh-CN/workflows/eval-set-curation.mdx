---
title: "评估集策划"
description: "构建一个具有代表性的翻译评估集，以发现和解决实际的质量问题。"
---

使用评估集来衡量您最关心的字符串的翻译质量。

## 选择文件格式

您可以将评估集存储为 JSON、JSONC 或 CSV 格式。

- 使用 JSON 或 JSONC 格式时，如果您需要嵌套结构和明确的元数据。
- use CSV when you want quick spreadsheet editing with columns like `id`, `source`, `targetLocale`, `context`, `reference`, `tags`, `bucket`, and `group`
- in CSV, separate multiple tags with semicolons (`ui;short;icu`)

## 选择具有代表性的覆盖范围

包括各种类型的字符串，以便您可以在不同内容形态中检测回归。

- 简短的 UI 字符串：按钮、标签、菜单项和简洁的错误文本
- 长文本：包括入职流程、帮助文本、法律条款和交易消息
- ICU 和复杂格式：包括复数规则、性别变体、选择语句以及日期或数字格式占位符
- 占位符和变量：例如 `{name}`, `%s` 或 `{{count}}` 这样的标记，这些标记必须在未更改的情况下保留。

## 尽量将上下文与每个用例保持一致。

针对每个案例，请存储一个稳定的ID，并提供足够的上下文信息供审核人员参考。

- 添加目标区域和源文本
- include screenshots, feature names, or intent notes in `context`
- 当您已经拥有可信的翻译时，添加可选的`reference`文本
- 使用标签对用例进行标记，并添加可选的“区域”或“组”，以便按区域对报告进行筛选。

## 长期保持质量

将评估集视为生产测试数据。

- - Review cycle: refresh set with UI or copy updates
- Quality maintenance: update over time, keep context close
- 删除不再与活动功能匹配的过时案例
- 保持简单、中等和困难字符串的平衡
- 重复运行相同的测试集，以便公平地比较模型或提示的变更。
