---
title: "GitHub Action 漂移检查"
description: "在 GitHub Actions 中运行 Hyperlocalise 本地化漂移检查。"
---

## 概览

当你希望在本地化偏移时使拉取请求检查失败时，请使用仓库根操作。

该操作运行 `hyperlocalise run --dry-run`，将 CLI 输出流式传输到工作流日志，并在启用时上传两个工件：

- `drift-report.json`
- `drift-summary.txt`

## 示例工作流程

```yaml
name: localization-drift

on:
  pull_request:

jobs:
  drift:
    runs-on: ubuntu-latest
    permissions:
      contents: read
    steps:
      - uses: actions/checkout@v4

      - uses: ./
        with:
          check: drift
          config-path: i18n.yml
          hyperlocalise-version: latest
          fail-on-drift: true
          upload-artifact: true
```

对于拉取请求检查，`check` 模式可以获取 GitHub 拉取请求差异并将其传递给 `hyperlocalise check --diff-stdin`。这会将结果和行内注释的范围限定为受支持的 `.json`、`.jsonc` 和 `.arb` 文件中已更改的翻译键。

```yaml
name: localization-check

on:
  pull_request:

jobs:
  check:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: read
    steps:
      - uses: actions/checkout@v4

      - uses: hyperlocalise/hyperlocalise@v1
        with:
          check: check
          config-path: i18n.yml
          github-diff: true
          fail-on-findings: true
```

外部仓库可以使用移动的 `v1` 引用来获取最新的稳定 v1 版本。如果你想要更严格的变更控制，请固定一个确切的标签。

```yaml
- uses: hyperlocalise/hyperlocalise@v1
  with:
    check: drift
```

## 输入

- `check`：检查名称。支持的值为 `drift` 和 `check`。
- `config-path`：配置文件的路径。默认值为 `i18n.yml`。
- `working-directory`：命令的工作目录。默认为 `.`。
- `hyperlocalise-version`：要安装的发布版本。默认为 `latest`。
- `fail-on-drift`：检测到漂移时使工作流失败。默认为`true`。
- `fail-on-findings`：在检测到检查发现时使工作流失败。默认值为`true`。
- `github-diff`：在 `check` 模式下处理拉取请求时，获取 GitHub PR diff 并将其传递给 `hyperlocalise check --diff-stdin`。默认为 `false`。
- `upload-artifact`：上传 JSON 和文本制品。默认为 `true`。

## 漂移检测

该操作将以下任一条件视为漂移：

- 干运行报告中的一个或多个可执行翻译任务
- 干运行报告中的一个或多个剪枝候选项

即使禁用了漂移强制执行，命令失败仍会使操作失败。

## 工件

`drift-summary.txt` 是快速分诊工件。它记录：

- 使用的配置路径
- CLI 退出代码
- 是否检测到漂移
- 受影响的目标路径
- 受影响的目标语言环境
- 有可用时受影响的条目键

如果 CLI 在写入 JSON 之前失败，操作在可能的情况下仍会写入一个最小的文本构件。
