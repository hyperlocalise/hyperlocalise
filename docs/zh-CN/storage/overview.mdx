## 存储层的作用

存储连接：

- 本地翻译文件,
- sidecar 溯源元数据
- 远程适配器 API。

## 支持的适配器

- [Crowdin](/storage/crowdin)
- [LILT AI](/storage/lilt)
- [Lokalise](/storage/lokalise)
- [短语](/storage/phrase)
- [POEditor](/storage/poeditor)
- [Smartling](/storage/smartling)

## 条目身份

每个翻译条目都通过以下方式进行标识：

- `key`
- `context`
- `locale`

## 侧边车元数据的存在原因是什么

您的面向用户的文本应保留在普通翻译文件中。 运营元数据（例如草稿与已整理状态）应保留在配套元数据文件中，以便翻译内容保持整洁。

## 拉取和推送生命周期

1. `run` 生成本地草稿 (`origin=llm`, `state=draft`)。
2. `sync push` sends local changes to remote storage.
3. 人工翻译人员在远程TMS中进行值维护。
4. `sync pull` 局部应用已整理的更改。

## 冲突规则

- 如果已启用，Pull 可以自动应用预先配置的透支功能。
- Push 默认情况下会保护经过选择的远程值。
- `--fail-on-conflict` 阻止无声漂移。
- `--force-conflicts` 是一个可选选项，用于处理风险较高的覆盖路径。

对于具体的冲突决策，请参阅 [conflicts](/troubleshooting/conflicts)。
