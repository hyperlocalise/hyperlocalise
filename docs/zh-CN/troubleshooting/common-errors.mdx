## `i18n.jsonc already exists; use --force to overwrite`

原因：正在运行 `init` 在配置已存在的情况下。

修复：

```bash
hyperlocalise init --force
```

## `storage config is required`

原因： `sync` 命令在未使用 top 的情况下使用-level `storage` 配置。

修复：添加 `storage.adapter` 和 `storage.config` in `i18n.jsonc`.

## `unsupported output format`

原因：错误 `--output` 值。

修复：

- `status`: 仅限 `csv`
- `sync`: `text` 或 `json`

## `unsupported file extension` 期间 `run`

原因：已映射的源文件或目标文件使用了本地解析器不支持的扩展名。

修复：

- 使用以下其中之一： `.json`, `.arb`, `.xlf`, `.xliff`, `.po`, `.md`, `.mdx`, `.strings`, `.stringsdict`, `.csv`
- 验证存储桶 `from` 和返回仅翻译后的文本，不要提供任何说明、标签、Markdown 或引号，除非翻译内容本身需要它们。 `to` 路径解析为预期的文件名

## 提供方身份验证失败

原因：缺少/无效的环境令牌。

修复：re-在同一 Shell 会话中导出 token，然后重新运行。
