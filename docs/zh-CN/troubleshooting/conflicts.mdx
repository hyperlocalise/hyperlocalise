## 拉取冲突

典型情况：本地精选值与远程值不同。

安全解决方案：

1. 运行 `sync pull` with `--output json`
2. 检查冲突条目
3. 确定事实来源
4. 使用更新后的本地/远程状态重新运行

`--apply-curated-over-draft` 仅在本地条目为草稿时适用。

## 推送冲突

典型情况：本地草稿与远程精选值不同。

安全解决方案：

1. 查看 JSON 格式的冲突报告
2. 保持默认的冲突失败行为
3. 使用 `--force-conflicts` 仅适用于已审核的例外情况

## 推荐默认值

- 保留 `--fail-on-conflict=true`
- 保留 `--dry-run=true` 直到报告审核完成
- 在 CI 中归档 JSON 报告以用于审计追踪
