---

## 拉取冲突

典型情况：本地精选值与远程值不同。

安全解析：

1. 运行 `sync pull`，使用 `--output json`
2. 检查冲突条目
3. 决定事实来源
4. 使用更新后的本地/远程状态重新运行

`--apply-curated-over-draft` 仅在本地条目为草稿时适用。

## 推送冲突

典型情况：本地草稿与远程整理值不同。

安全解析：

1. 在 JSON 中查看冲突报告
2. 保留默认冲突失败行为
3. 仅将 `--force-conflicts` 用于已审核的例外情况

## 推荐的默认值

- 保留 `--fail-on-conflict=true`
- 保持 `--dry-run=true`，直到报告经过审查
- 在 CI 中归档 JSON 报告以用于审计追踪
