## 最小配置模式

从一个 bucket 开始即可：

- `locales.source`：您的源语言 (例如 `en-US`)
- `locales.targets`: 一个目标语言，例如 `es-ES`
- `buckets.ui.files`: 把源路径模板映射到目标路径模板
- `llm.profiles.default`: 选择 provider 和 model

完整字段说明见 [i18n config](/zh-CN/configuration/i18n-config)。

在这个配置中，翻译内容来自 `packages/ui/lang`，输出到 `apps/web/lang`。如果省略 `groups`，Hyperlocalise 会自动让所有已配置 bucket 对所有目标语言执行。

## 生成的文件

运行 `hyperlocalise run` 之后：

- 翻译的目标文件在您配置的`to`路径下
- 项目根目录中的 `.hyperlocalise.lock.json`
- 横向存储的元数据文件，用于追溯，具体取决于您的存储布局
