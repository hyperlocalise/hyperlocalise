## 支持的占位符

- `{{source}}`: 源语言环境
- `{{target}}`: 目标区域设置
- `{{localeDir}}`：源语言环境输出留空，否则为目标语言环境

## 示例

### JSON 本地化文件

```jsonc
{
  "from": "packages/ui/lang/{{source}}.json",
  "to": "apps/web/lang/{{target}}.json"
}
```

### 带有的区域设置模板 `[locale]` token

```jsonc
{
  "from": "lang/en.json",
  "to": "lang/[locale].json"
}
```


### 使用 glob 的 Mintlify 递归映射

```jsonc
{
  "from": "docs/**/*.mdx",
  "to": "docs/{{localeDir}}/**/*.mdx"
}
```

这将保留源文件结构，并将翻译后的文件写入每个区域设置文件夹下相同的相对路径中。

With `source=en` 和目标 `fr`, `es`，和 `zh`:

- `target=en` 解决 `docs/{{localeDir}}/index.mdx` 到 `docs/index.mdx`
- `target=fr` 解决 `docs/{{localeDir}}/index.mdx` 至 `docs/fr/index.mdx`
- `target=es` 解决 `docs/{{localeDir}}/index.mdx` to `docs/es/index.mdx`
- `target=zh` 解决 `docs/{{localeDir}}/index.mdx` 到 `docs/zh/index.mdx`

当你的英文源页面直接位于以下位置下时，请使用此模式 `docs/` 并且翻译后的页面应写入 locale 文件夹中。

## 良好模式

- 保持源树和目标树相互独立。
- 按域名使用稳定的存储桶名称 (`ui`, `marketing`, `emails`).
- 每个文件映射只保留一项职责。

## 反制-patterns

- 写作 `to` 将路径返回到源目录中。
- 为多个存储桶重复使用相同的输出路径。
- 将区域设置令牌与广告混合使用-在模板之外进行 hoc 字符串替换。

## 快速检查

运行前：

1. 确认 `from` 为源区域设置解析，
2. 确认 `to` 可为每个目标区域设置创建，
3. 运行 `hyperlocalise run --dry-run` 用于检查计划的任务。
