`hyperlocalise` reads provider credentials from:

- 导出当前 shell 会话中的变量。
- `.env` and `.env.local` files in your current project directory.

优先级:

- 已经导出的 shell 变量保持不变。
- 对于尚未导出的变量，`.env.local`会覆盖 `.env`。

示例：

<CodeGroup>
```bash Export
export OPENAI_API_KEY="your-openai-api-key"
export AZURE_OPENAI_BASE_URL="https://<resource>.openai.azure.com/openai/v1"
export AZURE_OPENAI_API_KEY="your-azure-openai-api-key"
```

```bash .env.local
OPENAI_API_KEY="your-openai-api-key"
AZURE_OPENAI_BASE_URL="https://<resource>.openai.azure.com/openai/v1"
AZURE_OPENAI_API_KEY="your-azure-openai-api-key"
```
</CodeGroup>

## 安全指导

- 首选使用环境变量，而不是硬编码的令牌。
- 保留 `.env` 和 `.env.local` 不参与版本控制。
- 定期轮换和限制凭证，并将其限制为最小权限。

## 凭据管理标准，适用于本地和 CI 环境

请使用此运营模式，适用于所有远程存储适配器，包括Smartling、Lokalise以及内部自定义适配器。

### 本地开发

1. Commit only non-secret config (`*Env` variable names) to `i18n.jsonc`.
2. 将实际 token 存储在`.env.local`（仅限开发者机器）。
3. Add `.env` and `.env.local` to `.gitignore`.
4. 使用独立的开发和生产 TMS 凭证。

### CI 环境

1. 存储秘密在您的 CI 密钥管理器中（例如，GitHub Actions 密钥）。
2. 仅将密钥注入需要它们的同步任务中。
3. 将 CI 密钥映射到您适配器的配置中预期的环境变量。
4. 使用环境隔离的密钥和分支保护，以进行生产同步。

### 轮换和最小权限

- 按照固定时间表轮换 token（例如每 90 天），以及在团队成员离职时进行轮换。
- 为每个 TMS 工作区或项目创建专用 CI 服务用户。
- 限制每个令牌的使用，仅限于用于拉取/推送所需的最低 API 权限。
- 在删除之前的凭据之前，先运行一次“dry-run”同步，以验证轮换。

### 符合审计的日志记录策略

- 永远不要将原始的凭证信息打印到日志、调试输出、屏幕截图或问题评论中。
- 仅记录凭证来源名称 (`SMARTLING_USER_SECRET`, `LOKALISE_API_TOKEN`)，而不是值。
- Keep CLI output in machine-readable formats (`--output json` or `--output csv`) for audit trails.
- 按照您的保留策略，存储 CI 日志并同步工件。

## TMS 适配器安全检查清单

在启用具有写入功能的 CI 任务之前，请使用此清单。

- [ ] 所有适配器的密钥都通过环境变量提供，而不是内联配置。
- [ ] CI 使用专用服务凭证（不使用个人访问令牌）。
- [ ] 凭据范围限制为必需的项目和操作。
- [ ] 轮换计划已记录并由团队负责。
- [ ] 管道日志不暴露令牌值。
- [ ] Scheduled dry-run `sync pull` and `sync push` jobs run successfully.
- [ ] 需要在部署写入同步之前进行手动批准。

与 Smartling、Lokalise 以及基于存储适配器接口构建的自定义适配器一样，相同的检查清单也适用。
