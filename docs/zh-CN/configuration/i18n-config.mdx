## 文件概述

`i18n.jsonc` 控制区域设置、文件映射、模型配置文件和规则选择。

## 语言环境

- `locales.source`：源语言区域。
- `locales.targets`：目标区域设置列表。
- 可选 `locales.fallbacks`：每个区域设置的回退顺序。

## 存储桶

`buckets` 将源文件映射到目标输出模板。

每个文件映射使用：

- `from`: 源路径模板
- `to`: 目标路径模板

使用 `{{source}}`, `{{target}}`，以及 `{{localeDir}}` 在模板中。 `{{localeDir}}` 当 target 等于 source 时，解析为空段；否则解析为目标区域设置。


### CSV 文件映射模式

使用存储桶文件映射来建模 CSV 工作流。

每-locale 文件：

```json
{
  "from": "translations/en.csv",
  "to": "translations/[locale].csv"
}
```

共享多-locale 文件：

```json
{
  "from": "translations/catalog.csv",
  "to": "translations/catalog.csv"
}
```

用于共享的多-locale CSV 文件，保留一列键列和一列目标语言环境列，且该列 `run` 可以一致地更新。

## 组

`groups.<name>` 定义要一起处理的内容。

- `targets`：子集的 `locales.targets`
- `buckets`：存储桶名称列表

## LLM 配置文件

`llm.profiles.<name>` fields:

- `provider`: `openai`, `azure_openai`, `anthropic`, `gemini`, `bedrock`, `lmstudio`, `groq`，或 `ollama`
- `model`: 提供商模型 ID
- `prompt`: 翻译提示模板

提示变量：

- `{{source}}`
- `{{target}}`
- `{{input}}`

## 规则

`llm.rules` 按组选择配置文件。

- `priority`：较高者获胜
- `group`: 组名称
- `profile`: 配置文件名称

## 入门配置

```bash
hyperlocalise init
```

然后编辑生成的 `i18n.jsonc` 用于您的项目。

## 验证提示

- 将组目标保留在内部 `locales.targets`.
- 使组存储桶与之保持一致 `buckets` 键。
- 在各处保持配置文件名称一致 `profiles` 和返回仅包含翻译文本，不包含任何解释、标签、Markdown 或引号，除非翻译内容本身需要它们。 `rules`.
