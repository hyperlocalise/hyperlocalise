## 文件概览

`i18n.yml` 用来配置 locale、文件映射、模型 profile，以及可选的高级路由能力。为了兼容旧项目，`i18n.jsonc` 仍然受支持。

## 起始配置

```yaml
locales:
  source: en-US
  targets:
    - es-ES

buckets:
  ui:
    files:
      - from: packages/ui/lang/{{source}}.json
        to: apps/web/lang/{{target}}.json

llm:
  profiles:
    default:
      provider: openai
      model: gpt-5.2
```

这就是 `hyperlocalise init` 生成的模板结构。

如果省略 `groups`，Hyperlocalise 会自动使用一个隐式的 `default` 分组，其中包含：

- `locales.targets` 中的全部目标语言
- `buckets` 中的全部 bucket 名称

## Locales

- `locales.source`: 源语言。
- `locales.targets`: 目标语言列表。
- `locales.fallbacks`（可选）: 每个 locale 的 fallback 顺序。

## Buckets

`buckets` 用来把源文件映射到目标输出模板。

每条文件映射包含：

- `from`: 源路径模板
- `to`: 目标路径模板

模板中可使用 `{{source}}`、`{{target}}` 和 `{{localeDir}}`。当 target 与 source 相同时，`{{localeDir}}` 为空；否则为 target locale。

### CSV 文件映射模式

你也可以用 bucket 建模 CSV 工作流。

按 locale 拆分的文件：

```json
{
  "from": "translations/en.csv",
  "to": "translations/[locale].csv"
}
```

共享的多 locale 文件：

```json
{
  "from": "translations/catalog.csv",
  "to": "translations/catalog.csv"
}
```

对于共享的多 locale CSV，请保留一个 key 列和一个 target locale 列，以便 `run` 能稳定更新。

## Groups

`groups.<name>` 定义哪些内容会一起处理。

- `targets`: `locales.targets` 的子集
- `buckets`: bucket 名称列表

对新手配置来说，`groups` 是可选的。只有在你想按 locale 或文件集合拆分执行时才需要添加。

## LLM Profiles

`llm.profiles.<name>` 支持以下字段：

- `provider`: `openai`、`azure_openai`、`anthropic`、`gemini`、`bedrock`、`lmstudio`、`groq`、`mistral` 或 `ollama`
- `model`: provider 的 model id
- `system_prompt`（可选）: 显式 system message 模板
- `user_prompt`（可选）: 显式 user message 模板
- `prompt`（已弃用，可选）: 旧版 prompt 模板，会作为 system fallback

可用变量：

- `{{source}}`
- `{{target}}`
- `{{input}}`

## Rules

`llm.rules` 用来为 group 选择 profile。

- `priority`: 数值越大优先级越高
- `group`: group 名称
- `profile`: profile 名称

`llm.rules` 是可选的。如果没有匹配的规则，Hyperlocalise 会回退到 `llm.profiles.default`。

## Cache

`cache` 用来配置本地 L1/L2 行为。

- `enabled`（可选）: 启用缓存服务
- `db_path`（可选）: sqlite 缓存数据库路径
- `sqlite`（可选）: 连接调优
- `l1.max_items`（可选）: L1 淘汰上限
- `l2.enabled`（可选）: 启用 translation memory 查找和写入
- `l2.auto_accept_threshold`（可选）: `[0,1]` 范围内的模糊自动接受阈值，默认 `0.90`

TM 元数据枚举：

- provenance: `curated` | `draft` | `llm` | `tms` | `unknown`
- source: `run` | `sync_pull` | `sync_push` | `manual` | `import` | `legacy` | `unknown`

## 高级示例

```yaml
locales:
  source: en-US
  targets:
    - fr-FR
    - es-ES
  fallbacks:
    fr-FR:
      - es-ES
      - en-US

buckets:
  ui:
    files:
      - from: packages/ui/lang/{{source}}.json
        to: apps/web/lang/{{target}}.json
  docs:
    files:
      - from: docs/{{source}}/**/*.mdx
        to: docs/{{target}}/**/*.mdx

groups:
  app:
    targets:
      - fr-FR
      - es-ES
    buckets:
      - ui
  docs:
    targets:
      - fr-FR
    buckets:
      - docs

llm:
  profiles:
    default:
      provider: openai
      model: gpt-5.2
    docs_review:
      provider: openai
      model: gpt-5.2
      system_prompt: You are a senior technical translator.
      user_prompt: |-
        Translate from {{source}} to {{target}}.

        Keep Markdown and MDX syntax intact.

        {{input}}
  context_memory:
    provider: openai
    model: gpt-5.2
  rules:
    - priority: 100
      group: docs
      profile: docs_review

cache:
  enabled: true
  db_path: .hyperlocalise/cache/cache.sqlite
  sqlite:
    max_open_conns: 1
    max_idle_conns: 1
    conn_max_lifetime_seconds: 300
  l1:
    enabled: true
    max_items: 50000
  l2:
    enabled: true
    auto_accept_threshold: 0.9
  rag:
    enabled: false
    top_k: 5
```

## 生成 starter

```bash
hyperlocalise init
```

然后根据你的项目修改生成的 `i18n.yml`。

## 校验建议

- 如果省略 `groups`，所有 bucket 都会对所有目标语言执行。
- 保证 group 的 target 都在 `locales.targets` 中。
- 保证 group 的 bucket 名称与 `buckets` 中的键一致。
- 保证 `profiles` 和 `rules` 中的 profile 名称一致。
