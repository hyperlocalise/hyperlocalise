<Note>
此功能尚处于实验阶段。 标志、输出形状和行为可能会在不同版本之间发生变化。 请在受控的工作流程中使用，并在 CI 中固定版本。

报告模式和评分行为可能会随着评估工作流程的成熟而演变。
</Note>

使用`eval`命令，在具有代表性的评估集上衡量翻译质量，并在持续集成（CI）中强制执行质量阈值。

## 用法

```bash
hyperlocalise eval run --eval-set <path> [flags]
hyperlocalise eval compare --candidate <path> --baseline <path> [flags]
```

## eval 运行

在您的评估集上运行一个或多个实验变体。

### 这条命令的作用是什么

1. 从`--eval-set`加载 eval 数据集。
2. 从您选择的配置文件、提供商、模型和提示覆盖中扩展实验变体。
3. 执行针对每个实验变体的每个用例。
4. 使用内置启发式模式对每个翻译进行评分。
5. Optionally adds an LLM judge lane when you pass both `--eval-provider` and `--eval-model`.
6. Writes a JSON report when you pass `--output`.
7. 打印一份简洁的实验摘要表格。

### 标志

- `--eval-set`：评估数据集路径 (`.json`, `.jsonc`, `.csv`) (必填)
- `--profile`: 个人资料名称覆盖（可重复）
- `--provider`: 供应商覆盖（可重复）
- `--model`: model override (repeatable)
- `--prompt-file`：提示文件覆盖
- `--prompt`: inline prompt override (mutually exclusive with `--prompt-file`)
- `--eval-provider`：LLM 评估提供者
- `--eval-model`: model for LLM evaluation
- `--eval-prompt-file`: 评估提示文件覆盖
- `--eval-prompt`：内联评估提示覆盖（与`--eval-prompt-file`互斥）
- `--output`: JSON 输出报告路径

LLM evaluation mode turns on only when you pass both `--eval-provider` and `--eval-model`.

Your translation flags keep their current meaning. The `--eval-*` flags only control the judge lane.

在LLM评估模式下，参考翻译是可选的。如果存在，则评估者会将其用作风格和语气的指导。

### 摘要表字段

- `score`：实验的平均加权质量得分
- `pass_rate`: successful runs / total runs for the experiment
- `placeholder_violations`: 占位符完整性硬失败违规次数
- `latency_ms`: average latency for the experiment

### 示例

使用配置中的默认设置运行，并生成报告：

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.jsonc \
  --output ./artifacts/eval-report.json
```

运行一组配置文件和提供商/模型的覆盖选项：

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.csv \
  --profile default \
  --profile fast \
  --provider openai \
  --provider anthropic \
  --model gpt-4.1-mini \
  --model claude-sonnet-4-5
```

使用提示文件进行覆盖运行：

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.json \
  --prompt-file ./prompts/translation-eval.txt
```

使用 LLM 评估，并使用内联评估提示：

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.json \
  --eval-provider openai \
  --eval-model gpt-4.1-mini \
  --eval-prompt "Score translation quality from 0.0 to 1.0 and explain briefly." \
  --output ./artifacts/eval-report.json
```

使用 LLM 评估，并使用评判提示文件：

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.json \
  --eval-provider anthropic \
  --eval-model claude-sonnet-4-5 \
  --eval-prompt-file ./prompts/eval-judge.txt \
  --output ./artifacts/eval-report.json
```

## eval 比较

将候选报告与基准报告进行比较。

使用此命令在 CI 中，以防止质量退化。

The workflow stays the same: run `eval run` first, then run `eval compare`.

### 标志

- `--candidate`: candidate report JSON path (required)
- `--baseline`: baseline report JSON path (required)
- `--min-score`：最小候选得分
- `--max-regression`: maximum allowed score regression from baseline to candidate

### CI 行为

`eval compare` prefers the LLM aggregate score when both reports include a usable LLM judge aggregate. Otherwise, it falls back to the heuristic weighted score.

命令在以下情况下会退出时出错：

- candidate score is below `--min-score`, or
- score regression exceeds `--max-regression`.

### 示例

比较报告并仅打印摘要值：

```bash
hyperlocalise eval compare \
  --candidate ./artifacts/eval-candidate.json \
  --baseline ./artifacts/eval-baseline.json
```

Fail CI if the candidate score drops below `0.82` or regresses by more than `0.02`:

```bash
hyperlocalise eval compare \
  --candidate ./artifacts/eval-candidate.json \
  --baseline ./artifacts/eval-baseline.json \
  --min-score 0.82 \
  --max-regression 0.02
```
