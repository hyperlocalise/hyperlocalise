<Note>
此功能是实验性的。标志、输出形状和行为在不同版本之间可能会发生变化。请在受控的工作流程中使用，并在 CI 中固定版本。

冲突处理和同步协议可能会随着TMS集成的发展而演变。
</Note>

## 使用方法

```bash
hyperlocalise sync push [--config <path>] [flags]
```

## 安全默认行为

`sync push` defaults to dry-run mode and conflict failure.

## 标志

- `--config`: config path
- `--locale`: 目标地区(们)，可重复
- `--dry-run`：仅预览（默认 `true`）
- `--output`：`text` 或 `json`
- `--fail-on-conflict`：当检测到冲突时返回错误（默认：`true`）
- `--force-conflicts`: allow overwrite for mismatch cases where policy allows (default `false`)

## 示例

```bash
hyperlocalise sync push --locale es-ES --dry-run=false
```

请参阅 [冲突处理](/troubleshooting/conflicts)在使用前。`--force-conflicts`。
