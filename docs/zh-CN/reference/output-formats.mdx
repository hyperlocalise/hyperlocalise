---
title: "输出格式"
description: "run、status 和 sync 命令输出格式参考。"
---

## `run` 输出 (`text`)

关键摘要字段：

- `planned_total`
- `skipped_by_lock`
- `executable_total`
- `succeeded`
- `failed`
- `persisted_to_lock`

故障行格式：

```text
failure target=<path> key=<key> reason=<reason>
```

## `status` 输出 (`csv`)

标题列：

```csv
key,namespace,locale,status,origin,state
```

交互模式：

- `hyperlocalise status --output csv -i` 渲染一个用于人工分流的 TTY 仪表板。
- CSV 仍然是规范的机器格式-适用于 CI 和自动化的易读格式。

## `sync` output (`text` 或 `json`)

文本摘要字段：

- `action`
- `creates`
- `updates`
- `unchanged`
- `conflicts`
- `applied`
- `skipped`
- `warnings`

与 CI 检查集成时，请使用 JSON 输出。


## CSV 翻译文件

当 `run` 写入 `.csv` 翻译目标，对于新追加的键，输出行顺序是确定的 (按键排序)并且，现有行保持其原始顺序。
