## `run` output (`text`)

关键摘要字段

- `planned_total`
- `skipped_by_lock`
- `executable_total`
- `succeeded`
- `failed`
- `persisted_to_lock`

失败行格式：

```text
failure target=<path> key=<key> reason=<reason>
```

## `status` output (`csv`)

表头列：

```csv
key,namespace,locale,status,origin,state
```

交互模式：

- `hyperlocalise status --output csv -i` 渲染一个用于手动分诊的TTY仪表板。
- CSV 仍然是 CI 和自动化领域的标准机器可读格式。

## `sync` 输出 (`text` 或 `json`)

文本摘要字段：

- `action`
- `creates`
- `updates`
- `unchanged`
- `conflicts`
- `applied`
- `skipped`
- `warnings`

在与 CI 检查集成时，请使用 JSON 格式的输出。


## CSV 翻译文件

当`run`写入`.csv`的翻译目标时，新添加的键的输出行顺序是确定的（按键排序），而现有行会保持其原始顺序。
