---
title: "Sync"
description: "Pull and push translations between local files and remote storage adapters."
---

The `sync` command supports bidirectional synchronization through adapter-specific integrations.

## Usage

```bash
hyperlocalise sync [command]
```

Subcommands:

- `pull`: fetch remote translations into local files.
- `push`: send local translations to remote provider.

## `sync pull`

```bash
hyperlocalise sync pull [--config <path>] [flags]
```

Flags:

- `--config`: config path, default `i18n.jsonc`.
- `--locale`: target locale filter (repeatable).
- `--dry-run`: preview changes without applying.
- `--output`: `text` or `json`.
- `--fail-on-conflict`: return error on conflicts.
- `--apply-curated-over-draft`: allow curated remote values to replace local draft values.

## `sync push`

```bash
hyperlocalise sync push [--config <path>] [flags]
```

Flags:

- `--config`: config path, default `i18n.jsonc`.
- `--locale`: target locale filter (repeatable).
- `--dry-run`: preview changes without applying.
- `--output`: `text` or `json`.
- `--fail-on-conflict`: return error on conflicts.
- `--force-conflicts`: allow overwriting remote mismatches where policy permits.
