## Top-level usage

```bash
hyperlocalise [flags]
hyperlocalise [command]
```

## Available commands

- `completion`: generate shell autocompletion.
- `help`: show command help.
- `init`: write the latest `i18n.jsonc` template.
- `run`: generate local translations from source files.
- `status`: show translation status by locale.
- `sync`: pull or push translations with remote adapters.
- `version`: print CLI version.

## `run`

```bash
hyperlocalise run [--config <path>] [--dry-run]
```

Behavior:

- Loads and validates config.
- Plans translation tasks from configured group and bucket mappings.
- Skips tasks already tracked in `.hyperlocalise.lock.json`.
- Executes remaining tasks in parallel.
- Persists successful task completion into lock state.

Flags:

- `--config`: path to config, default `i18n.jsonc` in current directory.
- `--dry-run`: print plan only, no translations and no writes.
